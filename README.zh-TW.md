# Minecraft Mods Localizer

語言：[English](README.md) | 繁體中文

Minecraft Mods Localizer 是一個在瀏覽器中執行的 Minecraft 模組語言檔翻譯工具。它可以讀取模組 jar 與資源包中的 `assets/<namespace>/lang/*.json` 檔案，將偵測到的翻譯 key 顯示在可編輯的工作區，並在完成後匯出資源包或修補後的 jar 複本。

直接使用線上版本：

https://mc-localizer.triturbo.dev

這是一個靜態網頁應用程式。它使用 Vite 與 React 建置，正常使用時完全在瀏覽器中執行，不需要後端伺服器。

## 為什麼需要這個工具

Minecraft 模組常常有缺漏、不一致，或尚未完成的翻譯。手動處理通常需要解開 jar、比對不同語系的 JSON、追蹤 fallback 文字，最後還要重新打包，並避免不小心改壞原始模組檔。

Minecraft Mods Localizer 將這些步驟整理成一個審閱工作流程：

- 載入一個或多個模組 `.jar` 檔。
- 選擇性載入既有資源包作為翻譯來源。
- 選擇要產生的 Minecraft 語系。
- 比對來源、fallback、原版 Minecraft、手動、轉換，以及 LLM 產生的文字。
- 直接在瀏覽器中編輯單一 key。
- 將工作儲存成可重複使用的專案 patch。
- 完成後匯出資源包 zip 或修補後的 jar 複本。

原始模組 jar 不會被修改。

## 主要功能

- **Jar 與資源包掃描**：讀取本機 jar 與資源包中的 Minecraft 語言 JSON 檔。
- **依 namespace 審閱**：依 namespace 分組，讓每個模組可以分開檢查。
- **語系 fallback 鏈**：當目標語系缺少文字時，使用可設定的 fallback / 來源語系優先順序。
- **內建原版 Minecraft 參考**：包含 `minecraft` namespace 的 Minecraft 語系檔，作為唯讀參考資料。
- **手動 patch**：可以覆寫單一翻譯值，並儲存在專案 patch 檔中。
- **專案還原**：瀏覽器草稿儲存可以在本機還原未完成的工作。
- **詞彙提示**：在編輯器中顯示符合的術語，並用這些提示輔助 LLM 翻譯與 OpenCC 中文轉換。
- **OpenCC 中文轉換**：使用 `opencc-js` 在簡體中文、臺灣繁體中文、香港繁體中文之間轉換。
- **選用 LLM 翻譯**：可以從瀏覽器呼叫 OpenAI 相容的 chat-completions API endpoint。
- **匯出選項**：產生一般資源包 zip，或產生修補後的 jar 複本，不會改動原始檔案。

## 隱私與 API 金鑰聲明

這個專案以 local-first 為設計方向。

- 你選取的模組 jar 與資源包會由你的瀏覽器讀取。
- 專案 patch、資源包與修補後的 jar 複本都會在你的瀏覽器中產生，並下載到本機。
- 線上靜態版本不會將你的檔案上傳到伺服器進行掃描、編輯或匯出。
- 如果你使用選用的 LLM 翻譯功能，被選取的來源文字會直接從你的瀏覽器送到你設定的 API endpoint。
- 我們不會收集、儲存或記錄你的 API 金鑰。
- API 金鑰只會在你的瀏覽器中用於呼叫你設定的 OpenAI 相容 endpoint。
- API 金鑰不會寫入匯出的專案 patch 檔。
- 你的 LLM 供應商可能會依照該供應商自己的條款與紀錄政策接收請求資料。

請只在你信任的部署版本中輸入 API 金鑰。如果你想完整控制實際提供給使用者的應用程式，可以 clone 這個 repository 並自行部署。

## 自行部署

你可以 clone repository，建置靜態檔案，並部署到任何靜態網站託管服務。

```bash
git clone https://github.com/Tr1turbo/Minecraft-Mods-Localizer.git
cd Minecraft-Mods-Localizer
npm install
npm run build
```

正式站台會輸出到 `dist/`。

Vite build 使用相對資源路徑，因此 `dist/` 可以部署在網域根目錄或專案子路徑。常見選項包含 Cloudflare Pages、GitHub Pages、Netlify、Vercel static output、S3 相容物件儲存，或簡單的靜態檔案伺服器。

## 運作方式

這個 app 使用瀏覽器檔案 API 讀取使用者選取的本機檔案。模組 jar 與資源包會被檢查是否包含 Minecraft 語言檔，例如：

```text
assets/<namespace>/lang/en_us.json
assets/<namespace>/lang/zh_tw.json
assets/<namespace>/lang/ja_jp.json
```

偵測到的項目會合併成 catalog。對每個目標語系，app 會判斷最後使用的文字來自既有 jar 內容、匯入的資源包、設定的 fallback 語系、中文轉換結果、LLM 候選翻譯，或手動 patch。

### 詞彙提示

當 key 或來源文字符合已知 Minecraft 術語時，編輯器會在選取的項目旁顯示詞彙提示。這些提示可以輔助手動審閱，會被放入 LLM 翻譯 prompt，也會盡可能作為中文轉換的自訂字典使用。

### 中文轉換

中文語系轉換由 [`opencc-js`](https://github.com/nk2028/opencc-js) 提供。當 app 有中文來源值，且目標語系是另一個支援的中文變體時，可以自動產生轉換後的文字。

支援的轉換目標：

- `zh_cn`：簡體中文，使用 OpenCC `cn`
- `zh_tw`：臺灣繁體中文，使用 OpenCC `twp`
- `zh_hk`：香港繁體中文，使用 OpenCC `hk`

例如，如果模組已經有 `zh_cn` 文字，而你正在製作 `zh_tw`，app 可以用 OpenCC 轉換來源文字，並將結果標示為 `Converted`。轉換也會盡可能使用 app 詞彙表作為自訂 OpenCC 字典，讓 Minecraft 與模組術語在不同中文語系間保持一致。

匯出內容會根據 catalog 產生：

- **資源包匯出**：建立包含翻譯後 `assets/<namespace>/lang/*.json` 檔案的 zip。
- **修補 jar 匯出**：建立可下載的 jar 複本，內含修補後的語系檔。
- **專案 patch 匯出**：儲存手動編輯、LLM 候選翻譯、詞彙覆寫與設定，方便之後繼續工作。

## 部署設定

執行時預設值會從部署後 `index.html` 旁的 `app-config.json` 載入。Vite build 時會將 repository 中的預設檔 `public/app-config.json` 複製到 `dist/`。

`app-config.json` 是公開檔案，請不要把 API 金鑰放進去。

範例：

```json
{
  "schemaVersion": 1,
  "openai_api": {
    "base_url": "https://api.openai.com/v1",
    "model": "gpt-5.4-mini"
  },
  "app": {
    "packFormat": 34,
    "description": "Generated by Minecraft Mods Localizer",
    "llmBatchSize": 40,
    "llmConcurrency": 3,
    "llmReferenceMode": "en_us"
  }
}
```

部署設定只會作為第一次使用時的預設值。使用者在 app 中變更設定後，瀏覽器草稿會優先套用。

## 專案結構

```text
.
├── data/
│   └── curatedGlossary.json
├── minecraft/
│   └── lang/
├── public/
│   ├── app-config.json
│   └── assets/
├── src/
│   ├── app/
│   ├── components/
│   ├── features/
│   └── lib/
├── test/
├── index.html
├── package.json
└── vite.config.ts
```

- `src/lib/` 包含與框架無關的領域邏輯，例如掃描、patch、匯出、語系處理、詞彙匹配、LLM 呼叫與部署設定。
- `src/features/` 包含頁面與工作流程 UI。
- `src/components/` 包含共用 React UI 元件。
- `data/curatedGlossary.json` 包含內建詞彙表。
- `minecraft/lang/` 包含第三方 Minecraft 語系檔，作為唯讀參考資料。
- `test/` 包含 Vitest unit tests。
